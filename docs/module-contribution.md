# Proposer un module pédagogique

Ce guide décrit le minimum attendu pour proposer un module compatible T2DECODE.

## 1) Préparer le contenu

- Format accepté : Markdown ou JSON
- Objectif pédagogique explicite dès l’introduction
- Niveau visé indiqué (débutant, intermédiaire, avancé)
- Aucune dépendance à un service cloud

## 2) Structure recommandée

- `title` clair et spécifique
- `summary` en 2 à 4 lignes
- Sections courtes, progressives et orientées pratique
- Exercices/quiz quand pertinent

## 3) Vérifications avant soumission

- Contenu lisible hors connexion
- Commandes testées localement
- Liens externes non bloquants (bonus seulement)
- Terminologie cohérente et non ambiguë

## 4) Soumettre

1. Ouvrir une issue via le template “Proposition de module”
2. Décrire le besoin, le public cible et le format choisi
3. Joindre un exemple minimal du module (ou un extrait)
4. Ouvrir ensuite une PR selon [CONTRIBUTING.md](../CONTRIBUTING.md)
